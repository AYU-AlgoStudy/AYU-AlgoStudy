## 브렌치
- 개인은 **fork** 후, 매주 **주간 브랜치**로 작업합니다.
    - 브랜치명: `week01-닉네임` (예: `week01-zldzldzz`)
    - 다음 주는 `week02-닉네임` … (주 단위로 새 브랜치)
- Org 레포 `main`은 **직접 push 금지**, PR로만 반영

### 커밋 컨벤션 (Conventional Commits)
- 포맷: `type(scope): summary`
- 예시
    - `feat(boj/2206): 벽 부수고 이동하기 BFS 3차원 visited 추가`
    - `fix(pgs/네트워크): 방문 배열 초기화 누락 수정`
    - `refactor(boj/7576): 큐→ArrayDeque 교체`
    - `docs(week01): 오답노트 정리`

- PR: 주간 묶음 제출, 최소 1인 승인 + 통과 후 머지
- 단순 정답이 아니라 **실수 원인→대응 패턴**을 축적
- 모든 문제를 올리는 것이 아니라 틀리거나 정답을 확인 후 푼 문제만 공유
- 쉽게 해결한 경우 제외
